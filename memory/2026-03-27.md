# 2026-03-27

## 决策

- Android 侧 `ViewTreeStyle` 新增 `sourceTypographyUnit`、`platformFontScale`、`fontWeightRaw`，同时保留既有 `typographyUnit='dp'` 和归一化后的 `fontSize/lineHeight/letterSpacing`。
- Android 的 source 语义采用“主语义 + 保真补充”的策略：
  - `sourceTypographyUnit` 以 `sp` 作为主语义，代表文本字号的原始来源。
  - `platformFontScale` 取 `Configuration.fontScale`。
  - `fontWeightRaw` 透出 `Typeface` 的原始样式标记字符串，不再只保留 `bold/italic` 这种归一化结果。
- 对 Android 的文字采集，`textSize`、`lineHeight`、`letterSpacing` 来自不同运行时来源；因此在代码里保留注释说明，避免后续把单一 source unit 误解为三者完全同源。

## Android 模拟器稳定性专项

### 关键结论

- 本机存在双 adb 版本：
  - `/opt/homebrew/bin/adb` -> `36.0.2-14143358`
  - `/opt/homebrew/share/android-commandlinetools/platform-tools/adb` -> `37.0.0-14910828`
- 启动链路中存在离线历史设备（`emulator-5554 offline`），会影响 adb/forward 稳定性观测与部分 `-e` 命令。
- 网关 `Connection refused(errno 61)` 的直接触发点是 `28766->18766` forward 丢失；需要在启动后持续监测并自愈，而不只是一锤子创建。

### 本次改动

- `scripts/start-simulator-demo.sh`
  - 新增 `pin_adb_path`，在脚本进程内统一 adb 来源。
  - 新增 `forward_exists/ensure_forward`，驻留与 keepalive 阶段自动重建 forward。
  - 新增 `--keepalive-seconds`，用于启动成功后的持续保活与自愈。

### 验证摘要

- 集成链路（gateway + android）60 秒稳定观测：
  - `adb state=device` 连续 12 轮（每 5 秒）稳定。
  - `forward=ok` 连续稳定。
  - `GET http://127.0.0.1:28766/v2/ui-tree/inspector?deviceId=simulator-device` 返回 200。
  - `GET /v2/ui-tree/snapshot?...&refresh=1` 返回 200。
- 自愈回归：
  - 手工执行 `adb forward --remove tcp:28766` 后，5 秒内自动恢复。
- 轻量复核：
  - `bash -n scripts/start-simulator-demo.sh` 通过。
  - `./scripts/start-simulator-demo.sh --dry-run` 通过，解析到 `Neptune_API_34` 和 `emulator-5556`。
