# 2026-03-25

## 决策

- `neptune-sdk-android` 的 gateway discovery 结果在 payload host 命中 loopback/wildcard 时，`GatewayDiscoveryResult.endpoint.host` 回退到当前 candidate.host，`port` 继续优先使用 payload.port。
- loopback/wildcard 判定范围固定为：`127.0.0.1`、`localhost`、`::1`、`::`、`0.0.0.0`。
- 本地 HTTP 服务默认监听 `0.0.0.0`，通过 `createExportHttpServer(host = "...")` / `createPersistentExportHttpServer(host = "...")` 继续支持回环地址显式配置。
- 本地 HTTP 路由新增 `POST /v2/client/command`，当前仅支持 `ping`，响应 ACK JSON 只包含 `requestId`、`command`、`status`、可选 `message` 和 `timestamp`。
- 新增客户端注册会话：`ClientRegistrationSession` 启动时立即向 `POST /v2/clients:register` 注册，并按 `30s` 固定周期续约。
- 注册身份主键固定为 `platform + appId + deviceId`，`sessionId` 仅用于展示/诊断，不参与主键。
- SDK 侧保留 discovery，但不把 WS 作为主动回调模型的默认主链路。

## Android Emulator 排障

- 本机 `ANDROID_SDK_ROOT` 曾指向不存在的 `/Users/linhey/Library/Android/sdk`，这会直接导致 `emulator` 命令缺失。
- 本机真正可用的 SDK 根在 `/opt/homebrew/share/android-commandlinetools`，其中包含 `emulator/`、`platform-tools/`、`system-images/`。
- `avdmanager list avd` 能看到 `Neptune_API_34`，说明 AVD 已存在；`adb devices` 为空，说明问题在“未启动 emulator”，不是“AVD 不存在”。
- 新增 `neptune-sdk-android/scripts/start-simulator-demo.sh` 统一负责 SDK 探测、AVD 选择、emulator 启动、adb 在线等待、Gradle 安装与 Activity 启动。
- 脚本已通过 `bash -n`、`--dry-run` 和一次真实端到端执行验证；真实执行成功完成了 emulator 启动、`./gradlew :app:installDebug` 和 `adb shell am start`。
- 补充脚本稳定性策略：默认在 App 拉起后执行 30 秒驻留轮询（同一 `adb`），并在检测到 PATH `adb` 与 SDK `adb` 版本不一致时输出告警，避免用错 `adb` 导致“设备消失”的假阴性。
- 新增参数：`--residency-seconds N`（默认 30，`0` 可关闭驻留校验）和 `--keep-emulator-log`（成功时也保留 emulator 原始日志，便于排障复现）。
- 复测结果：`scripts/start-simulator-demo.sh --keep-emulator-log` 成功通过安装、启动和 30 秒驻留校验；`./gradlew smokeDemo` 通过。
