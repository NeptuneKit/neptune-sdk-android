# 2026-03-24

## 决策

- `sdk` 模块的 HTTP 导出层不再使用手写 JSON 拼接，改为 Jackson 树模型。
- `sdk` 模块的 HTTP 导出服务从 `NanoHTTPD` 迁移到 `Ktor Server + CIO`，保留原有路由与 JSON 行为。
- 查询参数解析保持本地实现，仅做数值提取与默认值回退。
- `GRDB 7+` 被记录为后续持久化层探索方向，但不进入本次 Kotlin/JVM HTTP 迁移范围。
- `sdk` 模块的导出队列持久化方案选用 `SQLDelight + SQLite`，保留默认内存模式，不采用 `Room`。
- 队列实现抽象为统一 `LogStore`，新增可选 SQLite 持久化模式，保持 `/v2/export/health`、`/v2/export/metrics`、`/v2/export/logs` 契约不变。
- 新增 `examples/smoke-demo` 可执行冒烟模块，通过 SDK 公开 API 发送日志并输出摘要；根构建增加 `smokeDemo` 任务，便于接入级验证。

## 风险结论

- 原手写 JSON 编码存在转义和结构一致性风险，已消除。
- 本次迁移的主要风险在于嵌入式 HTTP 服务行为回归，已通过真实起服测试覆盖健康检查与 405 错误响应。
- `GRDB 7+` 即便具备 Android 支持，也会把当前任务扩大为跨语言集成，不适合作为本次阻塞项。
- `Room` 对当前 `kotlin-jvm` 模块的 AndroidX 依赖过重，不利于 SDK 以 JVM 测试和宿主嵌入方式维护。
- 持久化队列的主要回归风险是游标稳定性、容量淘汰和 overflow 累计，已通过持久化重开测试覆盖。
